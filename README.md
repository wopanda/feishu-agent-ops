# Feishu Agent Ops

把飞书机器人接成 OpenClaw 多 Agent。  
支持添加机器人、巡检、根因排查和修复。

---

## 先说重点：你只管提供必要信息

前台只保留 2 个入口：
- 添加机器人
- 排查问题

你不需要判断这是“新增”还是“扩容”。
系统会先扫描当前环境，再给你最合适的接入方案或排查结论。

### 入口 1：添加机器人
最低只需要这些：

1. **appId**  
   去哪找：飞书开放平台 → 对应应用 → **凭证与基础信息** → App ID
2. **appSecret**  
   去哪找：同一页 → App Secret
3. **botName**  
   直接给我机器人名称 / 应用名称就行

### 这些信息现在默认可选
- **accountId**：OpenClaw 里的内部代号，**你不会起名也没关系，我可以帮你生成**
- **agentId**：只有你明确想绑定到某个已有 Agent 时才需要
- **roleName / model / isDefault**：没有就先不填
- **chatId**：只有你明确要做“群聊级绑定”时才需要

### 入口 2：排查问题
默认连上面这些都不用先给。

你只要直接说：

```text
这套多飞书机器人有问题，你先帮我检查。
```

我应该先做只读扫描，再决定后面要不要补信息。

## 去哪找信息

### 1）appId / appSecret 去哪找
路径：

**飞书开放平台 → 你的应用 → 凭证与基础信息**

你会看到：
- App ID
- App Secret

把这两个发我就行。

### 2）botName 去哪找
通常直接用：
- 飞书开放平台里的应用名称
- 或你在飞书里看到的机器人名字

### 3）accountId 是什么
这是 **OpenClaw 本地配置里的内部标识**，不是飞书官方字段。

比如可以叫：
- `rixin3`
- `research`
- `writer`
- `sales-bot`

如果你不想想名字，**我会按 botName 给你生成一个稳妥的 accountId**。

### 4）agentId 去哪找
只有你明确说“要绑定到某个已有 Agent”时才需要。

更稳的做法是：
**你先别找，我先扫描当前环境，再告诉你有哪些可复用的 agentId。**

### 5）chatId 什么时候才要
只有在这类需求里才需要：
- 不是私聊绑定
- 而是要把某个群固定绑定到某个 agent

如果你没提到群聊级绑定，默认先不问这个。

## 这版默认怎么引导

### 添加机器人
我默认应该这么做：

1. 先扫描当前环境  
2. 只补最少缺的信息  
3. 给最合适的接入方案和预览  
4. 你确认后再写配置  
5. 最后给验证路径

### 排障 / 巡检
我默认应该这么做：

1. 先只读扫描  
2. 判断是会话隔离、bindings、workspace、插件链还是版本兼容问题  
3. 先给根因判断  
4. 再决定要不要修

```text
- appId：cli_xxx
- appSecret：xxx

其他你帮我补。
```

```text
这套多飞书机器人有问题：

```

## 你会得到什么

### 添加机器人
- 我帮你补全缺的命名
- 我帮你判断绑已有 Agent 还是新建 Agent
- 我先给你变更预览
- 你确认后再落地

### 排障
- 兼容探测结果
- 当前现场扫描结果
- 根因判断
- 修复优先级
- 最小修复建议
