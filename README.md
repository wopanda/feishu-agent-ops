# Feishu Agent Ops

把你已经创建好的多个飞书机器人，接成 OpenClaw 多 Agent。  
支持新增、扩容、巡检、根因排查和修复。

---

## 先说重点：默认只要你给“必要信息”

之前前台收的信息太多，这版已经改了。

现在默认改成：

### 如果你是新增 / 扩容机器人
最低只需要这些：

1. **appId**  
   去哪找：飞书开放平台 → 对应应用 → **凭证与基础信息** → App ID
2. **appSecret**  
   去哪找：同一页 → App Secret
3. **botName**  
   直接给我机器人名称 / 应用名称就行

### 这些信息现在默认可选
- **accountId**：OpenClaw 里的内部代号，**你不会起名也没关系，我可以帮你生成**
- **agentId**：只有你想“绑定到某个已有 Agent”时才需要
- **roleName / model / isDefault**：没有就先不填
- **chatId**：只有你明确要做“群聊级绑定”时才需要

### 如果你是排障 / 巡检
默认连上面这些都不用先给。

你只要直接说：

```text
这套多飞书机器人有问题，你先帮我检查。
```

我应该先做只读扫描，再决定后面要不要补信息。

## 你现在可以直接这样用

### 场景 A：我想新增一个机器人
直接发：

```text
我想新增一个飞书机器人。
appId：xxx
appSecret：xxx
botName：日新调研
其他你帮我补。
```

### 场景 B：我想继续扩容
直接发：

```text
我已经有多个机器人了。
这是新机器人的 appId / appSecret / botName，帮我增量扩进去，不要覆盖旧配置。
```

### 场景 C：我不清楚 accountId / agentId 怎么填
直接发：

```text
我不会填 accountId / agentId，你先根据当前环境帮我判断，再给我预览。
```

### 场景 D：我这套环境有异常
直接发：

```text
这套多飞书机器人现在有异常，你先检查，不要直接乱改。
```

## 去哪找信息

### 1）appId / appSecret 去哪找
路径：

**飞书开放平台 → 你的应用 → 凭证与基础信息**

你会看到：
- App ID
- App Secret

把这两个发我就行。

### 2）botName 去哪找
通常直接用：
- 飞书开放平台里的应用名称
- 或你在飞书里看到的机器人名字

### 3）accountId 是什么
这是 **OpenClaw 本地配置里的内部标识**，不是飞书官方字段。

比如可以叫：
- `rixin3`
- `research`
- `writer`
- `sales-bot`

如果你不想想名字，**我会按 botName 给你生成一个稳妥的 accountId**。

### 4）agentId 去哪找
只有你明确说“要绑定到某个已有 Agent”时才需要。

更稳的做法是：
**你先别找，我先扫描当前环境，再告诉你有哪些可复用的 agentId。**

### 5）chatId 什么时候才要
只有在这类需求里才需要：
- 不是私聊绑定
- 而是要把某个群固定绑定到某个 agent

如果你没提到群聊级绑定，默认先不问这个。

## 这版默认怎么引导

### 新增 / 扩容
我默认应该这么做：

1. 先判断你是新增、扩容，还是排障  
2. 只补最少缺的信息  
3. 先给预览  
4. 你确认后再写配置  
5. 最后给验证路径

### 排障 / 巡检
我默认应该这么做：

1. 先只读扫描  
2. 判断是会话隔离、bindings、workspace、插件链还是版本兼容问题  
3. 先给根因判断  
4. 再决定要不要修

## 推荐的最小输入模板

### 新增一个机器人

```text
我想新增一个飞书机器人：
- appId：cli_xxx
- appSecret：xxx
- botName：日新调研
其他你帮我补。
```

### 增量扩容

```text
我已经有多机器人了，下面这个帮我增量加进去：
- appId：cli_xxx
- appSecret：xxx
- botName：内容助手
不要覆盖旧配置。
```

### 排障

```text
这套多飞书机器人有问题：
- 现象：某个 bot 不回复 / 串会话 / 路由错
- 先帮我检查，不要直接改
```

## 你会得到什么

### 新增 / 扩容
- 我帮你补全缺的命名
- 我帮你判断绑已有 Agent 还是新建 Agent
- 我先给你变更预览
- 你确认后再落地

### 排障
- 兼容探测结果
- 当前现场扫描结果
- 根因判断
- 修复优先级
- 最小修复建议

---

## 一句话原则

> 先让用户把事说出来，再只补必要信息；能我推断的，不先让用户填表。
