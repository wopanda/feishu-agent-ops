## 会话：多Agent / 通道 / Skills 共用边界梳理

### 用户当前困惑
- 用户明确表示对 OpenClaw 的“通道、Agent、多 Agent、子 Agent、Skills”之间关系看不清。
- 特别想弄明白：
  - 通道和 Agent 怎么配合；
  - 多 Agent 到底是不是多个“人格/工位”；
  - 哪些东西共用，哪些东西隔离；
  - Skills 会不会共用；
  - 书里相关章节应该如何统一讲清楚。

### 本轮梳理重点（需保留）
- 通道不是 Agent，通道更像消息入口 / 传输层；Agent 才是“大脑 / 工位 / 工作角色”。
- 多 Agent 本质上是多个隔离工作区 + 会话 + 认证状态的独立大脑，通过 bindings / route 按渠道、账号、对象分流。
- 子 Agent 更像从主 Agent 临时派出去的后台工人，不等于长期常驻多 Agent 架构。
- Skills 是否共用，取决于放在哪一层：
  - `<workspace>/skills`：该 Agent 私有；
  - `~/.openclaw/skills`：同机多个 Agent 共用；
  - 内置 Skills：全局基线。
- 章节上的最佳讲法应拆成：
  1. 第1章讲系统构成（通道/工作区/记忆/Skills/Agent）；
  2. 第2章讲团队化协作（主 Agent + 子 Agent + 多角色分工）；
  3. 第4章讲通道只是入口，不是大脑；
  4. 第5章讲 Skills 是能力包，并强调“私有 vs 共享”；
  5. 第12章讲治理：角色、路由、权限、门禁、责任链。
